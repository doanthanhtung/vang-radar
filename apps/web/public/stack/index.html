💡 Ý tưởng cốt lõi

Stack là gì? Vì sao cần Stack?

Stack là gì?
  • Cấu trúc dữ liệu tuyến tính (linear data structure)
  • Chỉ cho phép thao tác tại một đầu — gọi là Top
  • Hoạt động theo nguyên lý LIFO: Last In, First Out
Vì sao cần Stack?
  • Xử lý tuần tự theo thứ tự ngược (reverse order)
  • Quản lý ngữ cảnh gần nhất (most recent context)
  • Theo dõi trạng thái lồng nhau (nested state)

⚡ Các thao tác cơ bản

Mọi thao tác chính đều có độ phức tạp O(1)

Thao tác Ý nghĩa Độ phức tạp Ghi chú
push(x)Thêm phần tử x vào TopO(1)Tăng kích thước Stack
pop()Xóa & trả phần tử TopO(1)Lỗi nếu Stack rỗng
top() / peek()Xem phần tử Top, không xóaO(1)Không thay đổi Stack
isEmpty()Kiểm tra Stack có rỗngO(1)Trả về true / false
size()Trả số phần tử hiện tạiO(1)Đếm phần tử trong Stack
💡 Lưu ý quan trọng
  • pop() và top() trên Stack rỗng → Stack Underflow (lỗi runtime)
  • push() khi Stack đầy (Array cố định) → Stack Overflow
  • Luôn kiểm tra isEmpty() trước khi gọi pop() hoặc top()

⚙️ Cách cài đặt Stack

Hai cách phổ biến — cùng thao tác O(1), khác cách lưu trữ

📦 Bằng Mảng — top trỏ vị trí trên cùng

array 
int a[100];
int top = -1;  // -1 = rỗng

void push(int x) {
  a[++top] = x;
}

int pop() {
  return a[top--];
}

int peek() {
  return a[top];
}

bool empty() {
  return top == -1;
}
✅ Đơn giản, nhanh, bộ nhớ liên tục
❌ Giới hạn kích thước mảng (hoặc cần resize)

🔗 Bằng Linked List — head chính là Top

linked_list 
struct Node {
  int val;
  Node* next;
};

Node* head = NULL;

void push(int x) {
  head = new Node{x, head};
}

int pop() {
  int v = head->val;
  head = head->next;
  return v;
}

int peek() {
  return head->val;
}

bool empty() {
  return head == NULL;
}
✅ Kích thước linh hoạt, không giới hạn trước
❌ Tốn bộ nhớ con trỏ, cần quản lý cấp phát

📞 Ví dụ: Call Stack

Runtime quản lý lời gọi hàm lồng nhau — mỗi lần gọi push, return thì pop

Tình huống: Gọi main() → foo() → bar(). Mỗi hàm là một stack frame lưu biến cục bộ, địa chỉ return.

nested_calls.cpp 
void bar()  { /* in ra "C" */ }
void foo()  { bar(); }
int  main() { foo(); return 0; }
1
main() bắt đầu → push frame main
Call Stack (đáy → top) ┌────────┐ │ main │ ← Top └────────┘
2
main gọi foo() → push frame foo
┌────────┐ │ foo │ ← Top ├────────┤ │ main │ └────────┘
3
foo gọi bar() → push frame bar → in "C"
┌────────┐ │ bar │ ← Top ├────────┤ │ foo │ │ main │ └────────┘
4
bar return → pop · foo return → pop · main kết thúc

Stack trace khi crash = in ngược Call Stack. Đệ quy vô hạn → Stack Overflow.

↩️ Ví dụ: Undo/Redo & Điều hướng

Lưu trạng thái gần nhất — phù hợp editor, trình duyệt, Android

📝 Undo trong trình soạn thảo
1
Gõ "Hi" → push snapshot "Hi"
Undo Stack: [ "Hi" ]
2
Gõ tiếp "Hello" → push snapshot "Hello"
Undo Stack: [ "Hi", "Hello" ] ← Top
3
Nhấn Undo → pop "Hello", khôi phục "Hi"

Redo Stack push lại "Hello" nếu cần làm lại.

📱 Activity Stack (Android)
1
Mở màn hình A → push Activity A
Task Stack: [ A ]
2
Chuyển sang B, rồi C → push lần lượt
Task Stack: [ A, B, C ] ← Top
3
Nhấn Back → pop C, hiện B. Back tiếp → pop B, hiện A

Tương tự nút Back trình duyệt — URL/màn hình trước nằm dưới Top.

⚙️ Ví dụ: Parser & DFS

Stack xử lý cấu trúc lồng nhau trong biên dịch và duyệt đồ thị

🔍 Kiểm tra dấu ngoặc: { [ ( ) ] }
1
Gặp { [ ( → push lần lượt
Stack: [ '{', '[', '(' ] ← Top
2
Gặp ) → pop (, khớp ✓
Stack: [ '{', '[' ]
3
Gặp ] rồi } → pop hết, Stack rỗng → hợp lệ

Compiler dùng Stack tương tự khi parse if { ... while { ... } }

🗺️ DFS — tìm đường trong mê cung

Đi sâu nhất có thể, gặp ngõ cụt thì quay lui — Stack lưu các ô cần quay lại (hoặc dùng đệ quy = Call Stack ngầm).

1
Start → push (0,0). Đi xuống (1,0), (2,0)
Path Stack: [ (0,0), (1,0), (2,0) ]
2
Hết đường → pop (2,0), thử nhánh khác từ (1,0)

Mỗi lần pop = backtrack một bước — đúng tinh thần LIFO.

💡 Biểu thức: 3 + 4 * 5
  • Chuyển Infix → Postfix: 3 4 5 * + (Stack lưu toán hạng & toán tử)
  • Tính Postfix: gặp số push, gặp + pop 2 số, push kết quả
